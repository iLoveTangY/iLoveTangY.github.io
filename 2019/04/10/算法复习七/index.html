算法复习七

发表于 2019-04-10 | 更新于 2019-04-18

本篇复习（学习）了以下内容：

  • KMP算法
  • Manacher算法
  • BFPRT算法

这几个算法都是比较难的算法，以后要多多回顾。

KMP算法

原理

要理解一个算法首先需要理解这个算法是用来解决什么问题。KMP算法主要是用来对字符串匹配进行加速的。假设我们有ABAABAABAAAAA这个文本字符串和BAAAA这个模式字符串，要在文本字符串中找到模式字符串第一次出现的位置，暴力的字符串匹配过程如下：

  • 两个指针i和j分别指向文本字符串的开头和模式字符串的开头，另有一个指针k指向i；
  • 当k和j所指的字符相同时即k和j同时前进一步，如果j到了模式字符串的结尾，那么表示找到了一个匹配，此时返回i即为第一次出现的位置；
  • 如果k和j所指字符不同，那么i要前进一步，k要退回到新的i，j也要退回到模式字符串的开头，这样一直到找到了一个匹配或者到文本字符串的结尾。

假设文本字符串的长度为$N$，模式字符串的长度为$M$，那么暴力算法的复杂度为$O(N \times M)$。为了提高字符串匹配的速度，Knuth、Morris和Pratt三位大牛就发明了KMP算法，KMP算法的核心思想就是当匹配失败时，我们的指向文本字符串的指针不回退，只有指向模式字符串的指针做一定的回退操作。

为了讲清楚KMP算法，我们需要先了解两个概念：

  1. 以某个字符结尾的前缀和后缀的概念。

    以BAAAA这个模式串为例，第二个A（下标为2）的前缀有两个：B(str[0])和BA(str[0],str[1])，同样，后缀也是两个A(str[1])和BA(str[0],str[1])

  2. 某个字符的公共最长前后缀

    顾名思义，公共最长前缀就是某个字符的前缀和后缀相同并且最长的那个前缀和后缀。但是这里我们要加个限定条件，计算公共前缀和后缀时前缀和后缀不能包含最长的那个前缀和后缀，也即完全相同的那个前缀和后缀。比如，有字符串BABBABA，那么最后一个字符A的公共最长前后缀就应该是BAB，对于前缀来说就是str[0,1,2]，对于后缀来说就是str[3,4,5]。注意，不能是BABBAB，因为上面说了，不能包含完全相同的那个前缀和后缀。

在对字符串做匹配之前，我们先对模式串的每个字符计算出公共最长前后缀的长度，存入数组中，然后开始做匹配。还是以上面的字符串为例：文本串为BABBAABABBBBABBABA，模式串为BABBABA。

  1. 对模式串BABBABA计算每个字符的公共最长前后缀的长度存入一个next数组中，这里我们先人工计算出来，后面会讲如何快速的计算next数组。

    我们规定，next[0]=-1，这是一个边界条件；然后很明显，next[1]=0，因为我们规定了最长公共前后缀不能包括前后缀完全相同的那个；next[2]=0；next[3]=1，公共最长前后缀为B；next[4]=1，公共最长前后缀同样为B；next[5]=2，公共最长前后缀为BA；next[6]=3，公共最长前后缀为BAB。因此，我们的整个next数组为[-1, 0, 0, 1, 1, 2, 3]。

  2. 定义一个i指针指向文本串的开头，定义一个j指针指向模式串的开头，从这里开始，我们的i指针就只前进不后退了。如果i和j对应的字符匹配，则i和j都往后挪一位，如果失配，则j=next[j]，而i继续停留在当前位置，重复这个过程直到文本字符串结尾或者找到完全匹配。

这就是整个KMP算法匹配的流程。但是为什么这样做是对的呢？接下来通过一种不严格的方式来理解一下KMP算法。假设失配时的状态如下图：

上面一行为文本字符串，下面一行为模式串，分别在位置i和位置j处失配。s为位置j的字符的公共最长后缀对应的第一个字符。在这种失配的情况下，我们可以断定如果以i'到s之间的某个字符为起始位置来和模式串匹配是一定会失败的。为什么可以这么说呢？假设我们在位置i'到s之间有个s'可以和模式串中的0到j的部分匹配起来，那么在模式串中从s'对应的位置到j的位置一定和从0到s对应的位置之后的某个位置相同（s'到i的部分已经和模式串中s'到j的字符串匹配上了，并且我们已经假设s'到i的部分可以和模式串中从0到模式串中s对应的位置之后的某个位置的字符串能够匹配）。这样就找到了一个j的比公共最长前后缀更长的一个公共前后缀了，这可就不科学了。因此，我们可以断定s是第一个可能和模式串匹配上的位置，并且由于s到i已经和j的最长公共前缀匹配上了，所以我们可以直接将j定位到公共最长前缀的结束位置开始匹配。

接下来就剩最后一个问题了，如何能快速计算出next数组的值呢？接下来得分两种情况来讨论。

假设要计算i的next数组的值，那么可以分为以下两种情况：

  1. 模式串中位置i-1处的值和位置i处的字符的最长前缀的下一个字符相同：

    在上图中，位置i的字符为d，位置i-1处的字符为b，b的最长前缀的下一个字符也为b。那么我们可以肯定位置i处的字符的公共最长前后缀的长度为位置i-1处字符的公共最长前后缀的长度加一，即next[i]=next[i-1]+1。

  2. 模式串中位置i-1处的值和位置i处的字符的最长前缀的下一个字符不同：

    在这种c和b不相等的情况下，next[i]的值应该是多少呢？我们说应该是继续递归的比较c的公共最长前缀的下一个字符和b是否相等，如果相等则next[i]就是此时的公共最长前缀的长度加一，否则继续递归进行这个过程。为什么这样是正确的呢？因为我们求next[i]的过程实际上就是相当于拿位置i的某个后缀（文本串）去和模式串的某个前缀（模式串）匹配，根据next数组的定义，当失配时我们应该将模式串的指针j跳到next[j]的位置，这个过程实际上相当于模式串的自我匹配。

实现

理解了KMP算法的原理之后实现起来就比较简单了：

应用

给定字符串s，求解含有两个s的最短字符串。例如给定s='aba'，那么包含两个s的最短字符串就是ababa，只要在s后面添上ba即可。

假设字符串长度为i，我们可以求解next[i]位置的值（注意，是最后一个位置后一个位置），这样相当于求出了s[i]位置的公共最长前后缀，然后我们将s中最长公共前缀之后的部分都添加到字符串的末尾就形成了所求的字符串。例如，有字符串s=abctabc，我们可以求出next[7]=3，因此，我们把index=3以后的所有字符都加到已有字符的末尾，则得到abctabctabc。

Manacher算法

原理

给定一个字符串，求这个字符串的最长回文子串。例如给定字符串s=abcbata，则最长回文子串为abcba。我们先想想用这道题该怎么做。采用暴力法，我们可以以每个字符为回文中心，往外扩张，直到不是扩张到的字符串不是回文串。对每个字符这样做，求出最长的回文串。这样对奇回文（回文中心落在字符上）没有问题，但是没办法求出偶回文（回文中心落在两个字符中间）。为了解决这个问题，我们可以在字符串的每个字符中间和首尾插入一个特殊字符，这样就将所有的偶回文都变成了奇回文（可以自己画一画），然后再以每个字符为回文中心求出回文串。这样就得到了一个$O(N^2)$的算法，还是速度太慢。Manacher算法就是解决这个问题的一个$O(N)$的算法。

Manacher算法的预处理步骤还是跟暴力做法一样，每个字符之间插入一个特殊字符，只是对求回文串的长度这个过程进行了加速。整个算法流程如下：

  1. 首先定义一个辅助数组p，里面存放以字符串中每个位置的字符为中心的回文直径，比如#a#b#c#b#a#的回文直径就是11。
  2. 辅助数组中最大的元素除2就是给定字符串的最长回文子串。

    假设p[i]有个更长的回文直径为d+m == c+n，m和n为新增的部分，那么根据对称性，m范围内的字符串应该等于q范围内的字符串，并且n范围内的字符串应该等于p范围内的字符串，另外m范围内的字符串应该等于n范围内的字符串，这样由m==n，m==q，n==p可以推出p==q，也就是说i'应该有个更长的回文直径，这和我们的假设不符。

    我们可以断定p[i]等于R - i，跟上面一种情况类似，我们也可以断定更大的回文直径是不可能的。

实现

同样，理解了原理，实现起来也很简单：

应用

给定一个字符串str，在末尾添加最少的字符使其变成回文串。例如给定acbc，那么添加一个a即可。

了解了Manacher算法的求解过程，解这道题也就简单了。按照Manacher算法求解过程（当然要加上特殊字符），只要有一个回文中心的右边界包含了当前字符串的最右边界，那么把第一个字符开始到左边界为止的所有字符逆序加在字符串末尾即可。例如，有acbc，当我们回文右边界的回文中心来到b这个位置时回文右边界能到最右边的字符c，此时的回文左边界为最左边的c，那么我们把第一个字符开始到左边界位置的所有字符（这里只有一个字符a）逆序加在字符串末尾即可，得到acbca。

BFPRT算法

原理

给定一个数组，在其中求前K大或者前K小的数，也即Top K问题。

首先，第一个想法就是如果是求前K大的数，可以用最小堆。维护一个大小为K的堆，遍历数组，将每一个元素和堆顶元素比较，如果比堆顶元素大，则将堆顶元素弹出，数组当前元素入栈，这样最后能得到最大的K个元素。时间复杂度为$O(N\times logK)$。

如果数组内存中放的下，有没有更好的办法？有！回想下之前的文章算法复习二中，在实现快速排序中我们是先在数组中随机选一个数，然后大于它的放右边，小于它的放左边，等于它的放中间。我们可以利用这个过程来求前k大的数。在快速排序时划分好三个部分之后，我们判断左边数的个数M是否大于K，如果是，则继续在左边的数中递归进行划分，如果，不是则在右边的数中找K-M个数，这个过程同样可以递归进行。这个算法的平均时间复杂度为$O(N)$，但是！！如果是最坏情况（倒霉催的），每次选的数都是最大的数，那么这个算法就退化成了一个$O(N^2)$的算法了。。

BFPRT算法就是上天派来拯救这种最坏情况的。从上面的分析我们可以看到算法的关键在于如何选取那个切分点。BFPRT这五个人（没错，这就是五个人名的首字母）就想了一点办法来选取这个切分点。具体过程如下：

  1. 将N个元素划分为$N/5$组，每组5个元素
  2. 对每个组排序，找到这5个元素中的中位数
  3. 对于所有的中位数形成的新数组，找出新的中位数，这个数就是切分点。

为什么说这样选取比较好呢？看下图：

假设青色是我们找到的每个组的中位数，红色是我们找到的这些中位数中的中位数。总共的组数是$N/5$，那么组中的中位数比最后找到的中位数大的组数就应该是$N/5/2=N/10$，这些组中又起码有三个数比中位数大，那么总共大于等于中位数的元素个数就起码是$3N/10$。可以证明这个算法的最坏时间复杂度为$O(N)$，具体证明参见算法导论§9.3。
